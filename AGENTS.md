# Istudarne

Gamified quiz-based study platform: users import JSON question banks, answer
quizzes, track progress, share public quizzes, and chat in study groups.

The repository is a pnpm monorepo split into deployable apps and reusable
packages. The API is JSON-only. Frontend pages live in the web app.

## Stack

- **Package manager**: pnpm workspaces (`apps/*`, `packages/*`).
- **API runtime**: Cloudflare Workers (`apps/api/wrangler.jsonc`) with Hono and
  `@hono/zod-openapi` (OpenAPI 3.1, Swagger UI at `/docs`, spec at
  `/openapi.json`).
- **Frontend**: React 19 + React Router 8 in `apps/web/src/`, Tailwind v4, SSR
  through the web Worker in `apps/web/workers/app.ts`.
- **Data**: Neon/Postgres through Drizzle ORM (`packages/database`), R2
  (`QUIZ_FILES`) for JSON objects, Durable Object `StudyGroupChat` for group chat
  WebSockets, and Cloudflare Queue `istudarne-ai-jobs` for async AI jobs.
- **Auth**: `@istudarne/auth`, an ORM-agnostic Better Auth wrapper backed by the
  host database adapter.
- **Validation**: Zod v4 schemas in `@istudarne/contracts`.
- **i18n**: Paraglide messages in `apps/web/messages/`, compiled to
  `packages/utils/paraglide`.

## Commands

- `pnpm dev:api` - run the API Worker on port 8787 (secrets from
  `apps/api/.dev.vars`; see `apps/api/.dev.vars.example`).
- `pnpm dev:web` - run the React Router web app.
- `pnpm typecheck` - compile i18n, build API route declarations, then run every
  package/app `check` script.
- `pnpm build` - compile i18n, build API route declarations, then run every
  package/app `build` script.
- `pnpm lint` / `pnpm format` / `pnpm check` - run Biome from each app directory.
  Biome config intentionally lives only in `apps/api/biome.json` and
  `apps/web/biome.json` (tabs, double quotes, 100 cols).
- `pnpm db:generate` - generate Drizzle migrations from
  `packages/database/src/schema`.
- `pnpm db:migrate` / `pnpm db:studio` - run Drizzle against `DATABASE_URL`.
- `pnpm --filter @istudarne/api build:types` - emit flattened API route types
  into `apps/api/types/` for the web RPC client.
- `pnpm --filter @istudarne/api typegen` and
  `pnpm --filter @istudarne/web typegen` regenerate Cloudflare/React Router
  generated types. Never hand-edit `worker-configuration.d.ts`.

## Architecture

### API (`apps/api`)

Request path: `src/index.ts` (middleware stack) -> route handler -> service ->
repository -> query.

- `src/index.ts` wires request ID/logging, CORS allowlist via
  `ALLOWED_ORIGINS`, CSRF for cookie clients (skipped when `Authorization` is
  present), secure headers, ETag, pretty JSON, body limit, rate limiters, DI,
  Better Auth, session hydration, and OpenAPI.
- `src/routes/**` is one file per endpoint, exporting a `createRoute`
  definition and a `RouteHandler`-typed handler. `routes/index.ts` chains every
  `.openapi(route, handler)` call and exports `type ApiRoutes` for the web RPC
  client.
- `src/server/**` contains framework-free domain code: `services/`,
  `repositories/`, `queries/`, and the per-request DI container.
- Errors use `AppError`; `http/errorHandler.ts` maps app and Hono errors to
  `{ error }` JSON. Zod validation failures use the same shape via the Hono
  `defaultHook`.
- The API no longer serves landing/share/not-found HTML pages. Those belong in
  React Router under `apps/web/src/routes`.

### Web (`apps/web`)

- `src/routes.ts` is the route manifest. Route modules in `src/routes/` usually
  re-export page components from `src/pages/`.
- `src/lib/rpc.ts` imports `type ApiRoutes` from `@istudarne/api/routes`; this is
  a type-only coupling so contract changes fail at compile time without shipping
  API code to the browser.
- `@istudarne/utils/paraglide` is generated by the root `i18n:compile` script.
  Re-run it after editing `apps/web/messages/*.json`.
- Native/Capacitor builds should call the deployed API over HTTP and use Better
  Auth bearer transport; web builds can use the Better Auth session cookie.

### Packages

- `@istudarne/auth`: Better Auth factory with stateful database sessions,
  cookie + bearer transport, `bearer`, `multiSession`, and `openAPI` plugins,
  email sender abstraction, and optional host `databaseHooks`.
- `@istudarne/database`: Neon/Postgres Drizzle client, schema, and migrations.
  Auth persistence includes Better Auth's `user`, `session`, `account`,
  `verification`, and `rate_limit` tables plus the domain `users` table with
  `auth_user_id` as a unique FK to Better Auth's `user.id`.
- `@istudarne/contracts`: shared Zod schemas and TypeScript types for API
  contracts.
- `@istudarne/utils`: generated Paraglide output only.

## Auth Model

Better Auth owns identity, credentials, sessions, verification, password reset,
and rate-limit persistence.

- API host wiring is in `apps/api/src/auth.ts`.
- Better Auth endpoints are mounted at `/api/auth/*`; the reference UI/schema is
  available at `/api/auth/reference`.
- Sessions are stateful in Postgres via Better Auth tables.
- Web clients use Better Auth cookies. Native/Capacitor clients use
  `Authorization: Bearer` through the Better Auth bearer plugin.
- `BETTER_AUTH_SECRET` is required. `AUTH_BASE_URL` must match the public API
  origin so action links and cookies are scoped correctly.
- `databaseHooks.user.create.after` provisions the Istudarne domain profile in
  `users`.
- API middleware resolves `authUser`, `domainUser`, and `session`; application
  route handlers call `requireUser(c)` and receive the domain user.
- `GET /api/me` returns both `authUser` and `domainUser`.

## AI Jobs

`POST /api/ai/jobs` stores the payload in R2 (`ai/jobs/{id}/input.json`), inserts
an `ai_jobs` row, enqueues `{ jobId }`, and returns 202. Clients poll
`GET /api/ai/jobs/{jobId}`. The consumer in `apps/api/src/queue/aiJobs.ts`
calls OpenAI (`OPENAI_API_KEY`, model from `OPENAI_MODEL`) and writes the result
to R2. Keep the consumer `MAX_ATTEMPTS` in sync with `max_retries` in
`apps/api/wrangler.jsonc`.

## Rate Limiting

Cloudflare `ratelimits` bindings are configured in `apps/api/wrangler.jsonc`:
`AUTH_RATE_LIMITER` (10/min) and `AI_RATE_LIMITER` (5/min). The API middleware
keys limits by client IP + path and returns 429 JSON.

## Conventions

- Code, comments, docs, and commit messages: English only. UI strings go through
  Paraglide.
- Follow the local app Biome config: tabs, double quotes, semicolons, 100 cols.
- Tailwind classes stay inside components; merge conditional classes with
  `cx()` from `apps/web/src/lib/classes.ts`.
- Static web assets live in `apps/web/public/`; imported React images live in
  `apps/web/src/assets/`.
- API routes stay RAG-friendly: OpenAPI spec, `/llms.txt`, and Markdown quiz
  export (`/api/quizzes/:id/export?format=markdown`).
- Keep app code and package code decoupled at runtime. Type-only imports from
  API route declarations are fine.

## Deploy Checklist

1. API secrets: `BETTER_AUTH_SECRET`, `DATABASE_URL`, optional
   `RESEND_API_KEY`, and optional `OPENAI_API_KEY`.
2. API vars: set production `AUTH_BASE_URL`, `ALLOWED_ORIGINS`, `EMAIL_FROM`,
   and `OPENAI_MODEL` in `apps/api/wrangler.jsonc` or the Cloudflare dashboard.
3. Create the queue: `wrangler queues create istudarne-ai-jobs`.
4. Create/configure the R2 bucket `istudarne-quiz-files`.
5. Apply Postgres migrations with `pnpm db:migrate`.
6. Deploy API from `apps/api` and web from `apps/web`.
